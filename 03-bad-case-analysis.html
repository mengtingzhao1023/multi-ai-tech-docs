Bad Case分析与优化

从错误中学习：分类错误的案例分析、根因定位与迭代优化

版本: v1.0 | 更新日期: 2026-04-10 | 状态: 已完成
---

一、文档概述

1.1 背景

任何AI系统都无法做到100%准确。Bad Case（错误案例）是系统优化的重要数据来源。通过系统性地收集、分析Bad Case，可以持续改进整理引擎的效果。

1.2 目标

  • • 建立Bad Case分类体系
  • • 设计用户反馈收集机制
  • • 实现闭环优化流程
  • • 构建标签纠错监控系统
---

二、典型Bad Case分类

2.1 错误类型分析

Case 1: 多主题混合对话

内容："今天学习了Python的列表推导式，顺便整理了下周的工作计划，晚上还要准备孩子的家长会..."

错误分类：仅标记为"学习"，忽略了"工作"和"生活"

根因：单标签分类模型无法处理多主题内容

Case 2: 专业术语歧义

内容："Java的Spring框架和床垫的Spring支撑系统有什么区别？"

错误分类：标记为"生活/家居"，实际应为"技术/编程"

根因：关键词"Spring"被误判，缺乏上下文理解

Case 3: 口语化表达

内容："那个啥，帮我看看这个bug咋回事，急！"

错误分类：标记为"其他"，未识别出技术内容

根因：口语化、缺乏关键词，规则引擎失效

2.2 错误类型统计

错误类型 占比 根因 优化难度
多主题混合 35% 单标签限制 中
术语歧义 25% 上下文缺失 低
口语化表达 20% 规则覆盖不足 中
标签缺失 15% 召回率不足 低
其他 5% - 高

三、优化方案与效果

3.1 针对性优化

优化1: 多标签分类

方案：从单标签改为多标签分类，允许一个对话属于多个类别

效果：Case 1问题解决，多主题识别准确率提升至92%

优化2: 上下文感知

方案：引入上下文窗口，结合前后对话内容判断语义

效果：Case 2问题解决，歧义识别准确率提升至88%

优化3: 口语化Prompt

方案：优化AI Prompt，明确告知处理口语化、非正式表达

效果：Case 3问题解决，口语内容识别准确率提升至85%

3.2 优化效果追踪

优化项 优化前 优化后 提升
多主题识别率 65% 92% +27%
歧义识别率 72% 88% +16%
口语内容识别率 68% 85% +17%

四、用户反馈收集机制

4.1 设计目标

Bad Case 的数据来源依赖于用户的主动反馈。我们在产品界面中嵌入多维度反馈入口，确保用户能够方便地报告错误。

4.2 反馈入口设计

反馈入口

  • • 对话卡片上的"标签有误"按钮
  • • 整理结果页的"纠错"悬浮按钮
  • • 批量操作中的"标记错误"选项
  • • 导出前的"确认标签"弹窗

反馈信息

  • • 错误类型（分类/标签缺失/多余）
  • • 用户期望的正确标签
  • • 纠错原因（可选填）
  • • 用户ID和时间戳

4.3 反馈收集实现

// 用户反馈收集器
class UserFeedbackCollector {
  // 收集标签纠错反馈
  async collectTagCorrection(dialogId, originalTags, correctedTags, reason) {
    const feedback = {
      type: 'TAG_CORRECTION',
      dialogId,
      originalTags,
      correctedTags,
      reason,
      timestamp: Date.now(),
      userId: getCurrentUserId()
    };
    
    // 存入反馈队列
    await this.feedbackQueue.push(feedback);
    
    // 实时分析是否为高频错误模式
    if (this.isHighFrequencyPattern(originalTags, correctedTags)) {
      this.triggerAlert(feedback);
    }
    
    return feedback.id;
  }
  
  // 判断是否为高频错误模式
  isHighFrequencyPattern(original, corrected) {
    const key = `${original.join(',')}->${corrected.join(',')}`;
    const count = this.patternCounter.get(key) || 0;
    this.patternCounter.set(key, count + 1);
    return count >= 3; // 3次以上视为高频
  }
}

5.2 闭环优化流程

// Bad Case闭环优化系统
class BadCaseManager {
  constructor() {
    this.badCases = [];
    this.optimizationLog = [];
    this.feedbackCollector = new UserFeedbackCollector();
    this.monitor = new TagCorrectionMonitor();
  }

  // 1. 从用户反馈中收集Bad Case
  async collectFromUserFeedback(feedback) {
    const badCase = {
      id: generateId(),
      content: await this.getDialogContent(feedback.dialogId),
      prediction: { tags: feedback.originalTags },
      groundTruth: { tags: feedback.correctedTags },
      timestamp: new Date(),
      source: 'USER_FEEDBACK',
      category: this.categorizeError(feedback.originalTags, feedback.correctedTags)
    };
    this.badCases.push(badCase);
    return badCase.id;
  }

  // 2. 定期分析（结合监控数据）
  async analyzeWeekly() {
    const monitorReport = await this.monitor.generateWeeklyReport();
    
    return {
      total: this.badCases.length,
      topPatterns: monitorReport.topPatterns,
      optimizationSuggestions: this.monitor.generateOptimizationSuggestions(monitorReport.topPatterns)
    };
  }

  // 3. Prompt优化与A/B测试
  optimizePrompt(errorPattern) {
    const newPrompt = this.adjustPrompt(basePrompt, errorPattern);
    const testResult = this.abTest(newPrompt, basePrompt);
    
    if (testResult.improvement > 0.05) {
      this.deployPrompt(newPrompt);
    }
  }
}
