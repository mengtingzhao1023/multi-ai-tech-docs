Memory与MCP

让AI拥有记忆：短期记忆、长期记忆与模型上下文协议

版本: v1.0 | 更新日期: 2026-04-10 | 状态: 已完成
---

一、文档概述

1.1 背景

Memory系统让整理引擎能够记住用户偏好、历史决策，实现个性化整理。MCP（Model Context Protocol）则提供标准化的工具调用接口，让整理能力可以被其他AI应用使用。

1.2 目标

  • • 设计三层Memory系统
  • • 实现个性化整理引擎
  • • 接入MCP协议
  • • 构建工具生态
---

二、Memory系统设计

2.1 三层架构

Memory系统让整理引擎能够记住用户偏好、历史决策，实现个性化整理。

短期记忆（Short-term Memory）

当前会话中的上下文信息，随会话结束而清除。

  • • 当前整理任务的对话列表
  • • 用户实时反馈（如纠错）
  • • 中间计算结果缓存

长期记忆（Long-term Memory）

持久化存储的用户偏好和历史数据，跨会话可用。

  • • 用户标签使用习惯
  • • 历史纠错记录
  • • 整理偏好设置
  • • 对话关联图谱

外部记忆（External Memory）

超出当前系统的知识库，通过检索获取。

  • • 统一标签库
  • • 领域知识库
  • • 同义词词典

2.2 Memory对比

类型 存储位置 生命周期 用途
短期记忆 内存 会话级 临时上下文
长期记忆 IndexedDB 永久 用户偏好
外部记忆 向量数据库 永久 知识检索
---

三、MCP协议接入

3.1 MCP简介

MCP是Anthropic提出的开放协议，用于标准化AI模型与外部工具的交互。

MCP核心价值

  • • 标准化接口：统一AI与工具的通信协议
  • • 可扩展性：轻松接入新工具和数据源
  • • 安全性：受控的工具访问权限
  • • 可移植性：一次开发，多处使用

3.2 MCP服务器实现

// MCP服务器实现
class MCPServer {
  constructor() {
    this.tools = new Map();
    this.resources = new Map();
  }

  // 注册工具
  registerTool(name, handler, schema) {
    this.tools.set(name, { handler, schema });
  }

  // 处理MCP请求
  async handleRequest(request) {
    switch (request.method) {
      case 'tools/list':
        return this.listTools();
      case 'tools/call':
        return this.callTool(request.params);
      default:
        throw new Error(`未知方法: ${request.method}`);
    }
  }

  // 列出可用工具
  listTools() {
    return Array.from(this.tools.entries()).map(([name, { schema }]) => ({
      name,
      description: schema.description,
      parameters: schema.parameters
    }));
  }
}
