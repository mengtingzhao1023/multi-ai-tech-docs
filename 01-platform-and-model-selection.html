技术文档 01

平台选择与AI引擎模型选型

为什么选这4个平台？AI整理引擎如何用免费模型实现核心功能？

版本: v1.0 | 更新日期: 2026-04-10 | 状态: 已完成
---

一、文档概述

1.1 背景

在开发多平台AI对话收藏自动整理工具时，需要解决两个核心问题：一是选择支持哪些AI对话平台，二是如何设计AI整理引擎的模型选型策略。本文档详细阐述这两个关键决策的技术依据和实现方案。

1.2 目标

  • • 明确4个平台选择的逻辑和依据
  • • 设计"免费为主、付费可选"的模型分层架构
  • • 实现模型路由与自动降级机制
---

二、对话来源平台选择

2.1 选择逻辑

在选择支持的平台时，我们分析了国内AI对话工具的市场份额和用户画像。最终选择豆包、Kimi、千问、DeepSeek这4个平台，因为它们分别代表了不同的使用场景和用户群体，合计覆盖国内AI对话用户的80%以上。

2.4 为什么不选其他平台？

文心一言、讯飞星火等平台虽然也有用户，但要么API开放程度不足，要么用户群体与上述4个平台高度重叠。从ROI角度，优先支持这4个平台可以覆盖绝大多数用户需求。

三、AI整理引擎模型选型

3.1 核心策略

我们的AI整理引擎采用"免费模型处理90%场景，付费模型处理10%复杂场景"的策略。这样既能保证核心功能零成本运行，又能在需要时提供高质量的输出。

3.2 四层模型架构

第一层：本地规则引擎（免费）

处理60%场景

基于用户自定义规则和关键词匹配，零成本、极速响应

关键词匹配 正则规则 TF-IDF提取

第二层：免费大模型API（免费）

处理30%场景

利用各平台免费额度，处理需要理解的复杂场景

通义千问免费版
分类、标签生成
Kimi免费版
长文本摘要

第三层：低成本大模型（低成本）

处理10%场景

复杂推理和高质量输出，成本控制在最低

DeepSeek V3 - 推理能力强，价格仅为GPT-4的1/10

扩展层：用户自定义API（用户付费）

可选

提供OpenAI、Claude等付费API接入接口，用户自行配置，平台不收费

四、技术实现：模型路由与降级

4.1 设计目标

为了实现"免费为主"的策略，我们设计了一个智能路由系统，根据任务复杂度和模型可用性自动选择最优方案。

4.2 核心代码实现

// AI整理引擎核心：模型路由与降级策略
class AIEngine {
  constructor() {
    // 模型优先级配置（从免费到付费）
    this.modelChain = {
      classification: [
        { name: 'local-rules', cost: 0, timeout: 100 },
        { name: 'qianwen-free', cost: 0, timeout: 2000 },
        { name: 'deepseek-v3', cost: 0.001, timeout: 3000 }
      ],
      summary: [
        { name: 'kimi-free', cost: 0, timeout: 5000 },
        { name: 'deepseek-v3', cost: 0.001, timeout: 4000 }
      ]
    };
  }

  async process(task, content) {
    const chain = this.modelChain[task];
    
    // 逐级尝试，失败自动降级
    for (const model of chain) {
      try {
        const result = await this.callModel(model.name, content);
        console.log(`✓ 任务完成：使用 ${model.name}，成本 ${model.cost}`);
        return result;
      } catch (error) {
        console.log(`✗ ${model.name} 失败，降级到下一级...`);
        continue;
      }
    }
    
    throw new Error('所有模型均不可用');
  }

  // 智能选择：根据内容复杂度选择模型
  async smartSelect(content) {
    const complexity = this.assessComplexity(content);
    
    if (complexity < 0.3) {
      return 'local-rules';  // 简单内容用本地规则
    } else if (complexity < 0.7) {
      return 'qianwen-free'; // 中等复杂度用免费模型
    } else {
      return 'deepseek-v3';  // 复杂内容用付费模型（低成本）
    }
  }

  assessComplexity(content) {
    // 基于内容长度、专业术语密度、句子复杂度评估
    const length = content.length;
    const techTerms = (content.match(/API|算法|模型|代码/g) || []).length;
    return Math.min((length / 1000 + techTerms / 10) / 2, 1);
  }
}

4.3 设计亮点

  • • 自动降级：免费模型不可用时，自动切换到低成本模型，保证服务可用性
  • • 成本可控：90%场景使用免费模型，整体成本趋近于零
  • • 扩展性强：用户可自定义接入付费API，平台不限制

五、思考过程：选型的权衡与决策

Q1: 为什么不用GPT-4作为主力？

GPT-4效果确实最好，但成本太高（$0.03/1K tokens）。我们的场景是高频、大批量的对话整理，如果全部用GPT-4，每月成本可能上千元。而国内免费模型的效果已经足够好，性价比更高。

Q2: 免费模型额度用完了怎么办？

我们设计了多平台轮询机制：通义千问额度用完就切Kimi，Kimi用完就切DeepSeek（低成本）。同时本地规则引擎可以处理大部分简单场景，减少对API的依赖。

Q3: 为什么不全部用本地模型？

本地模型（如Transformers.js）虽然免费，但准确率低、需要GPU、模型体积大。对于分类这种需要理解语义的复杂任务，云端大模型效果更好。我们采用"本地+云端"的混合策略，平衡成本和效果。

Q4: 用户自定义API的意义是什么？

这是产品开放性的体现。有些用户已经有OpenAI API Key，或者对输出质量有极高要求（如法律、医疗场景）。提供自定义接口，让用户自己选择，既满足了高端需求，我们又不承担成本。
