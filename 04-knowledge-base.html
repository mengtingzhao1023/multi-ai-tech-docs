知识库与标签体系

三层分类架构：规则+AI混合的标签系统设计

版本: v1.0 | 更新日期: 2026-04-10 | 状态: 已完成
---

一、文档概述

1.1 背景

标签体系是AI整理引擎的核心组件。一个设计良好的标签体系需要兼顾结构化与灵活性，既要满足统一分类的需求，又要支持用户的个性化表达。

1.2 目标

  • • 设计三层标签体系架构
  • • 实现规则+AI混合的标签分配
  • • 构建知识图谱发现关联
  • • 支持用户自定义扩展
---

二、标签体系架构

2.1 三层架构设计

一级：系统标签

平台预设，不可修改

工作 学习 生活

二级：领域标签

用户自定义，AI辅助

编程 设计 管理

三级：动态标签

AI自动生成，高频提取

Python AI工具 效率

2.2 标签层级关系

层级 来源 可修改 使用场景
系统标签 平台预设 否 基础分类维度
领域标签 用户+AI 是 专业领域细分
动态标签 AI提取 部分 内容关键词

三、规则引擎设计

3.1 设计目标

规则引擎用于处理明确的、可模式化的分类场景，作为AI分类的补充和前置过滤。

3.2 核心实现

// 用户自定义标签规则引擎
class TagRuleEngine {
  constructor() {
    this.rules = {
      // 关键词匹配规则
      keywords: {
        "AI工具": ["ChatGPT", "Claude", "Copilot", "Midjourney"],
        "编程开发": ["代码", "算法", "Python", "JavaScript", "API"],
        "产品设计": ["PRD", "原型", "用户调研", "交互设计"]
      },
      
      // 正则匹配规则（更灵活）
      regex: {
        "会议记录": /会议|例会|周会|复盘|同步/i,
        "技术方案": /方案|设计|架构|技术选型|评审/i,
        "问题排查": /问题|故障|排查|解决|修复|bug/i
      },
      
      // 组合规则（多条件）
      composite: [
        {
          name: "AI编程",
          conditions: [
            { type: "keyword", value: "代码" },
            { type: "keyword", value: "AI" }
          ],
          operator: "AND"
        }
      ]
    };
  }

  // 执行规则匹配
  match(text) {
    const matchedTags = [];
    
    // 1. 关键词匹配
    for (const [tag, keywords] of Object.entries(this.rules.keywords)) {
      if (keywords.some(k => text.includes(k))) {
        matchedTags.push({ tag, source: 'keyword', confidence: 0.9 });
      }
    }
    
    // 2. 正则匹配
    for (const [tag, pattern] of Object.entries(this.rules.regex)) {
      if (pattern.test(text)) {
        matchedTags.push({ tag, source: 'regex', confidence: 0.85 });
      }
    }
    
    // 3. 组合规则
    for (const rule of this.rules.composite) {
      if (this.evaluateCompositeRule(rule, text)) {
        matchedTags.push({ tag: rule.name, source: 'composite', confidence: 0.95 });
      }
    }
    
    return this.deduplicate(matchedTags);
  }
}

四、知识图谱构建

4.1 设计目标

基于标签共现关系，构建知识图谱，发现隐藏的知识关联，支持标签推荐和智能分类。

4.2 核心实现

// 知识图谱构建
class KnowledgeGraph {
  constructor() {
    this.nodes = new Map(); // 标签节点
    this.edges = new Map(); // 关联边
  }

  // 从对话中提取标签共现关系
  buildFromConversations(conversations) {
    for (const conv of conversations) {
      const tags = conv.tags;
      
      // 记录节点
      for (const tag of tags) {
        if (!this.nodes.has(tag)) {
          this.nodes.set(tag, { name: tag, count: 0, conversations: [] });
        }
        this.nodes.get(tag).count++;
        this.nodes.get(tag).conversations.push(conv.id);
      }
      
      // 记录边（标签共现）
      for (let i = 0; i < tags.length; i++) {
        for (let j = i + 1; j < tags.length; j++) {
          const edgeKey = [tags[i], tags[j]].sort().join('-');
          if (!this.edges.has(edgeKey)) {
            this.edges.set(edgeKey, { source: tags[i], target: tags[j], weight: 0 });
          }
          this.edges.get(edgeKey).weight++;
        }
      }
    }
  }

  // 发现相关标签
  findRelatedTags(tag, limit = 5) {
    const related = [];
    
    for (const [key, edge] of this.edges) {
      if (edge.source === tag) {
        related.push({ tag: edge.target, weight: edge.weight });
      } else if (edge.target === tag) {
        related.push({ tag: edge.source, weight: edge.weight });
      }
    }
    
    return related
      .sort((a, b) => b.weight - a.weight)
      .slice(0, limit);
  }

  // 推荐标签（基于已有标签）
  recommendTags(existingTags) {
    const candidates = new Map();
    
    for (const tag of existingTags) {
      const related = this.findRelatedTags(tag, 3);
      for (const { tag: relatedTag, weight } of related) {
        if (!existingTags.includes(relatedTag)) {
          candidates.set(relatedTag, (candidates.get(relatedTag) || 0) + weight);
        }
      }
    }
    
    return Array.from(candidates.entries())
      .sort((a, b) => b[1] - a[1])
      .slice(0, 3)
      .map(([tag]) => tag);
  }
}
